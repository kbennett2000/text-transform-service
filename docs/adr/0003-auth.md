# ADR 0003: Auth — optional shared-secret header, default off

**Status:** Accepted
**Date:** 2026-07-13

> Transcribed from DESIGN §2 (ADR-0003).

## Context

The service is LAN-only and credential-free by posture. Some deployments may still want a
low-effort guard against accidental cross-service calls, without standing up real identity.

## Decision

Support an **optional shared-secret header, default off**:

- If env `TRANSFORM_API_KEY` is **set**, every `/v1/*` request must carry
  `X-Transform-Key: <value>` or receive **401**.
- If **unset**, no auth is enforced (LAN posture).
- **`/health` is always unauthenticated**, regardless of the key.

## Consequences

- Zero-config on a trusted LAN; a single env var turns on a coarse guard.
- Not real authn/authz — one shared secret, no rotation story in v1. Adequate for the
  threat model (a private LAN), explicitly not for internet exposure.
- `/health` staying open keeps liveness probes and the Scriptorium orchestrator's health
  checks simple.
- Implemented in cycle T7 as a FastAPI dependency; the 401 body uses the standard error
  envelope (DESIGN §4).
